# 積読＆ドラゴンズ - 企画概要

## コンセプト
積読消化をRPGのように楽しくやれるWebアプリケーション

## 基本的な仕組み

### 本 = 敵
- 積読したら本を登録
- 本は敵（ドラゴン）に見立てられる
- ページ数 = 敵のHP（ページ数が多いほど強敵）

### 読書 = 戦闘
- 読み始めたら戦闘開始
- 読み進めたページ数分、敵のHPを削る
- 削ったHPに応じて経験値獲得
- HP削りきったら討伐完了、ボーナス経験値

### スキルシステム
- 本の内容に応じたスキルに経験値が入る（例：DB本→DBスキル）
- スキルはユーザーが選択/自由入力で設定
- 経験値が貯まるとレベルアップ
- スキル間の階層構造なし（フラット）

### 再読
- 本は再度戦うことも可能

## ターゲット
- 技術書に限らず一般書籍も対象
- 基本的に一人で使う想定

## 提供価値
- 数値が増えて進捗が可視化される → 学習促進
- 自分が何を成長できたかを可視化できる

---

## MVP機能

1. **本の登録・管理**
2. **読書進捗の記録（戦闘）**
3. **経験値・スキルシステム**
4. **ステータス確認**

## 技術スタック

| 項目 | 選定 |
|------|------|
| フロントエンド | React + TypeScript（Vite） |
| ホスティング | Cloudflare Pages |
| API | Cloudflare Workers + Hono |
| DB | DynamoDB |
| 認証 | Firebase Auth（Google, Twitter） |
| 画像ストレージ | Cloudflare R2 |

## 主要な設計決定

### 認証
- SNS認証を導入
- 理由：複数デバイスでの利用を想定

### 戦闘システム
- ドラクエ風のメッセージウィンドウ演出
- ユーザーからの一方的な攻撃（敵からの反撃なし）
- 「あなたの攻撃！ 30のダメージ！」のような表示

### 敵（ドラゴン）の画像
- デフォルト画像を1種類用意してスタート
- 実装は複数種類を選択可能な構造にしておく
- 将来的にはAI生成でオリジナルドラゴン

### スキル表示
- MVP時点ではシンプルなリスト表示
- 後でグラフィカルな表示を検討
